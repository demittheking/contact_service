## Как запустить проект

### 1. Клонирование репозитория
git clone https://github.com/ваш-username/contact-service.git
cd contact-service

### 2. Настройка виртуального окружения
bash
# Создаем виртуальное окружение
python -m venv venv

# Активируем
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
### 3. Установка зависимостей

pip install -r requirements.txt

### 4. Настройка переменных окружения
Создайте файл .env в корне проекта и скопируйте в него:

# Django
SECRET_KEY=ваш-секретный-ключ
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

# OpenAI API (оставить пустым, если нет ключа)
OPENAI_API_KEY=

# Email (настройте для отправки писем)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password
DEFAULT_FROM_EMAIL=your-email@gmail.com

# Rate Limiting
RATE_LIMIT_REQUESTS=5
RATE_LIMIT_PERIOD=60

# CORS
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://localhost:8000

### 5. Применение миграций
bash
python manage.py migrate

### 6. Запуск сервера
bash
python manage.py runserver
Сервер будет доступен по адресу: http://127.0.0.1:8000

### 🛠 Стек технологий
#### Backend
Язык: Python 3.10+

Фреймворк: Django 4.2 + Django REST Framework (DRF)

#### Библиотеки:

django-cors-headers — настройка CORS

python-dotenv — управление переменными окружения

drf-spectacular — OpenAPI документация (в процессе интеграции)

#### AI
OpenAI API — анализ тональности комментариев и генерация автоматических ответов

Fallback-алгоритм — встроенный анализатор на основе ключевых слов (при недоступности OpenAI)

#### Хранение данных
SQLite — для миграций Django (можно заменить на PostgreSQL)

Файловая система — для логов и rate limiting

#### Дополнительно
SMTP — отправка email-уведомлений через Django EmailBackend

### Архитектура
Проект построен на слоистой архитектуре с разделением ответственности:

contact_service/
├── config/                  # Настройки проекта
│   ├── settings.py          # Конфигурация Django
│   └── urls.py              # Корневые маршруты

│   │   └── urls.py          # Маршруты API
│   │
│   ├── services/            # Бизнес-логика (Services)
│   │   ├── ai_service.py    # AI-интеграция
│   │   └── email_service.py # Отправка писем
│   │
│   └── core/                # Инфраструктурный слой

### Паттерны проектирования
Controller-Service — разделение логики обработки запросов и бизнес-логики

Dependency Injection — сервисы внедряются через импорты

Decorator Pattern — rate limiting реализован через декоратор

Middleware — логирование запросов на уровне фреймворка

### Почему выбраны эти технологии?
Django — предоставляет всё необходимое из коробки (ORM, админка, middleware)

DRF — упрощает создание REST API и сериализацию

Файловая система — не требует настройки БД для тестового задания

OpenAI — простой и мощный API для AI-функций

### Реализация API
Базовый URL
text
http://localhost:8000/api/
Эндпоинты
POST /api/contact
Отправка сообщения через контактную форму.

Заголовки:

Content-Type: application/json

Тело запроса:

Поле	Тип	Обязательное	Описание
name	string	Да	Имя пользователя (2-100 символов)
phone	string	Да	Номер телефона в формате +7XXXXXXXXXX
email	string	Да	Email пользователя
comment	string	Да	Текст сообщения (1-1000 символов)
# Пример запроса:

json
{
    "name": "Дмитрий Карасёв",
    "phone": "+79040731166",
    "email": "dkarasev42@gmail.com",
    "comment": "Здравствуйте! Хочу заказать разработку сайта для своего бизнеса."
}
# Пример успешного ответа (201 Created):

json
{
    "status": "success",
    "request_id": "a1b2c3d4",
    "message": "Ваше сообщение успешно отправлено!",
    "sentiment": "positive",
    "auto_reply": "Здравствуйте, Иван! Спасибо за ваш интерес. Мы свяжемся с вами в ближайшее время для обсуждения деталей."
}
# Пример ошибки валидации (400 Bad Request):

json
{
    "errors": {
        "phone": ["Неверный формат телефона. Используйте +7XXXXXXXXXX"]
    }
}
# Пример превышения лимита (429 Too Many Requests):

json
{
    "error": "Слишком много запросов. Попробуйте позже."
}
# Валидация
name — минимум 2 символа, максимум 100

phone — проверка формата через регулярное выражение

email — проверка через EmailField Django

comment — минимум 1 символ, максимум 1000

# Обработка ошибок
400 — ошибка валидации данных

429 — превышение rate limiting

500 — внутренняя ошибка сервера (логируется)

# AI-интеграция
Используемые AI-инструменты
OpenAI API (ChatGPT)

Анализ тональности — определение эмоциональной окраски комментария (positive/negative/neutral)

Генерация ответа — автоматическое создание персонализированного ответа на обращение

### Промпты
# Для анализа тональности:

Ты анализируешь тональность комментария. 
Ответь строго одним словом: positive, negative или neutral.
# Для генерации ответа:

Ты - менеджер по продажам. Ответь вежливо и предложи помощь. 
Ответ должен быть кратким (2-3 предложения).
Graceful Fallback
Если OpenAI API недоступен (ошибка сети, отсутствие ключа, региональные ограничения), сервис автоматически переключается на встроенный алгоритм:

### Fallback-анализ тональности:

Использует словари позитивных и негативных слов

Подсчитывает количество совпадений

Определяет тональность на основе сравнения

Fallback-ответ:

Возвращает null (автоответ не генерируется)

Сервис продолжает работу без ошибок

# Хранение данных
### Логирование
Формат: текстовый (с временными метками)

Расположение: logs/app.log

Содержание:

Все входящие запросы (метод, путь, IP)

Коды ответов

Ошибки с полным стектрейсом

AI-запросы и их результаты

Настройка: через Django LOGGING в settings.py

### Rate Limiting
Тип: файловое кеширование

Расположение: data/rate_limits.json

Формат:

json
{
    "rate_limit_192.168.1.1": ["2024-01-01T12:00:00", "2024-01-01T12:01:00"]
}
Логика:

Ключ: rate_limit_{IP}

Значение: список времен запросов в формате ISO

Очистка: удаление записей старше RATE_LIMIT_PERIOD секунд

Лимит: RATE_LIMIT_REQUESTS запросов за период
